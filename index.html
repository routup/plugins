routup ecosystem

Plugins for routup

A small, focused set of plugins for the routup HTTP router — request parsing, decorators, rate limiting, OpenAPI generation, metrics, i18n. Each ships independently, follows the same Web-Standards conventions, and weighs only what it needs to.

Browse all pluginsView on GitHub

Bundles

One install, the things almost every API needs.

@routup/basic

Body, cookie, and query parsing in one router.use() call.

  • Wraps body + cookie + query
  • Re-exports each helper
  • Pairs with decorators
Read docs →

Request & response

Parse what comes in. Serialize what goes out.

@routup/body

JSON, URL-encoded, raw, text, blobs, and decompressed streams.

  • Size limits per type
  • Transparent gzip/brotli
  • Cached on event.store
Read docs →

@routup/cookie

Cookie parsing + Set-Cookie serialization with all attributes.

  • Full SameSite / Secure / HttpOnly support
  • setRequestCookies for middleware
  • unsetResponseCookie helper
Read docs →

@routup/query

Query-string parsing into a typed map, including nested keys.

  • ?foo[bar]=1 nesting
  • Cached parse
  • Tree-shakeable helpers
Read docs →

@routup/assets

Static-file serving with cache-control, fallback, and SPA support.

  • SPA fallback
  • Pre-scanned metadata
  • Per-pattern ignores
Read docs →

Routing patterns

Different ways to define handlers.

@routup/decorators

Class-based controllers with TypeScript decorators.

  • @DController + method decorators
  • Parameter injection
  • OpenAPI source for swagger-preset
Read docs →

API documentation

Generate and serve OpenAPI documents.

@routup/swagger

Generator (V2 + V3) plus Swagger UI mountable on any path.

  • OpenAPI v2 + v3
  • Reads decorator metadata
  • Inline Swagger UI
Read docs →

@routup/swagger-preset

The @trapi/metadata preset that decodes routup decorators.

  • Maps @D* decorators
  • Used by the swagger generator
  • Customizable on top of buildPreset()
Read docs →

Operations

Throttle, observe, translate.

@routup/rate-limit

Per-client request quotas with pluggable stores.

  • Sliding window
  • Custom keyGenerator / skip
  • In-memory default store
Read docs →

@routup/rate-limit-redis

Redis-backed Store for distributed rate-limiting.

  • Shared counters across pods
  • ioredis connection or string
  • Drop-in store replacement
Read docs →

@routup/prometheus

HTTP metrics + a /metrics scrape endpoint for Prometheus.

  • Built-in uptime + requestDuration
  • prom-client default registry
  • Custom metrics auto-exported
Read docs →

@routup/i18n

Per-request locale resolution and a translator helper, built on ilingo.

  • ilingo Store interface
  • Accept-Language detection
  • Interpolation + plural support
Read docs →
